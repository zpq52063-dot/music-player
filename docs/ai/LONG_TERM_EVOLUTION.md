# Long Term Evolution

> Phase 11 — 长期演进路线图 | 2026-05-24
>
> **重要:** 以下内容仅为未来展望，当前不要实现。

---

## 演进原则

1. **当前优先** — P0/P1 先完成才考虑演进
2. **渐进式** — 每次只演进一个方向，不颠覆式重构
3. **向后兼容** — 新功能不能破坏现有架构
4. **用户驱动** — 基于实际使用需求做演进决策

### Phase 14: 多设备同步

**状态:** P2 (未来)

```
目标: 同一用户的多个设备间同步数据

技术方案:
  - Supabase Realtime (WebSocket)
  - 播放进度同步
  - 歌单同步
  - 喜欢歌曲同步
  - 冲突解决: Last-Write-Wins + 服务端时间戳

当前阻塞:
  - 需要真实 Supabase 数据连接 (Phase 12)
```

---

### Phase 15: NAS 本地音源

**状态:** P3 (未来)

```
目标: 用户自有的 NAS/本地音乐文件作为音源

技术方案:
  - WebDAV / SMB 协议
  - 文件扫描 + 元数据解析 (ID3 tags)
  - 流式传输
  - 本地索引数据库

当前阻塞:
  - 需要自建代理服务
  - 大文件传输需优化
```

---

### Phase 16: 自建音源

**状态:** P3 (未来)

```
目标: 部署自有音源代理，消除对非官方API的依赖

技术方案:
  - 自建 API 代理服务器
  - 缓存常用歌曲
  - 限流 + 鉴权
  - Docker 化部署

当前阻塞:
  - 需要服务器资源
  - 需要维护成本评估
```

### Phase 17: 本地歌词库

**状态:** P3 (未来)

## 演进优先级时间线

```
现在 (Phase 11)  → AI原生工程体系
近期 (Phase 12)  → 真实数据 + Vercel部署
短期 (Phase 13)  → 测试 + CI/CD
中期 (Phase 14-15) → 多设备 + NAS音源
长期 (Phase 16-20) → 自建 + 家庭 + 新设备
```

---

## 不演进的方向

以下明确不做：

- ❌ 商业化 (保持私用)
- ❌ 社交网络 (保持私密)
- ❌ 广告系统 (破坏体验)
- ❌ Web3/区块链 (无场景)
- ❌ AI 推荐引擎 (过度设计)
- ❌ 多语言国际化 (仅中文)
- ❌ Android 适配 (仅Apple生态)
