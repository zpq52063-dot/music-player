## 长期演进建议

1. **自建音源代理服务** — 部署在自有服务器上，消除对非官方API的依赖
2. **NAS 本地音源** — 用户自有的音乐文件作为音源
3. **WebDAV 接入** — 支持个人云存储中的音乐文件
4. **缓存第一策略** — 播放过的歌曲全部缓存到 IndexedDB，减少对外部API的依赖

## 当前 Provider 依赖链

```
用户搜索/播放
  → ProviderManager.execute()
    → NeteaseProvider (P0, 优先)
      ↓ 失败
    → QQProvider (P1)
      ↓ 失败
    → KuwoProvider (P2)
      ↓ 失败
    → MockProvider (P99, 最终兜底)
```

**关键保障:** 只要 MockProvider 存在且 ProviderManager 正常工作，系统可降级但不会完全不可用。
